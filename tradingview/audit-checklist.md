# 用户审计清单（按需读取）

用此清单验证AI是否严格按缠论推导，缺3条以上说明分析不够严格：

## 结构完整性
- [ ] 4H：分型→笔→线段→中枢→走势类型→MACD方向→结论
- [ ] 1H：同上完整流程 + 明确中枢ZD/ZG
- [ ] 15M：同上 + 买卖点类型 + 背驰比较对象 + 区间套

## 定义可审计
- [ ] 中枢有明确ZD/ZG价位（来自用户提供或估算标注"未确认"）
- [ ] 买卖点类型有明确依据（哪个中枢、哪段离开、哪段回拉）
- [ ] 背驰有明确比较对象（哪两段、围绕哪个中枢）

## 诚实性
- [ ] 背驰判断标注了"视觉判断"还是"数值确认"
- [ ] 看不清的地方标注了"未确认"
- [ ] 多义性情况列出了多种解读

## 一致性
- [ ] 中枢位置前后一致
- [ ] 买卖点判断前后一致
- [ ] 方向结论前后一致

---

## 标准追问模板

**追问中枢：**
> "你说的这个中枢，由哪三段次级别走势重叠构成？ZD/ZG怎么算出来的？"

**追问买卖点：**
> "你说这里是X买/X卖，请只证明这个买卖点，不要给交易建议。哪段是离开中枢？哪段是回拉？为什么算不回中枢？"

**追问背驰：**
> "你说出现背驰，比较的是哪两段？围绕哪个中枢？是视觉判断还是数值确认？数值来自哪里？"

**一致性检验（选做）：**
> 同一组图用不同问法再问一次（"只给结构，不给建议"），核心结论应保持一致。
