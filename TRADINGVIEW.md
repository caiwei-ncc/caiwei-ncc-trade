# GBP/USD 日内交易执行协议

## 零、文件定位与依赖关系

### 本文件是什么
交易执行协议。定义 GBP/USD 日内交易的风控规则、入场/出场策略、分析内容要求、输出规范和用户话术模板。

### 本文件不是什么
不是缠论定义手册。所有缠论底层概念（分型、笔、线段、中枢、走势类型、背驰、三类买卖点、区间套、多义性等）的判定标准，统一查阅 `tradingview/chan-definitions.md`。

### 依赖文件

| 文件 | 用途 | 何时读取 |
|------|------|---------|
| `tradingview/chan-definitions.md` | 缠论判定定义（最终版） | **每次分析前强制读取** |
| `tradingview/audit-checklist.md` | 分析审计清单 | 用户说"审计"时读取 |
| `tradingview/chan-theory-lessons-*.md` | 原典课程文档 | 按需追溯 |
| `TOOLS.md` 交易执行纪律部分 | 12 步流程、输出排版模板、模型切换规则 | 系统自动加载 |

### 分工原则
- **本文件**：管"分析什么"——每个级别分析哪些要素、风控怎么算、入场出场怎么定
- **chan-definitions.md**：管"怎么判"——每个缠论概念的严格定义、判定规则、常见误判
- **TOOLS.md 交易执行纪律**：管"怎么执行和排版"——12 步流程、Opus 准入、固定输出模板、图片处理、状态播报
- 若某条规则已在 definitions 层被更清楚定义，本文件只保留执行层引用（如"防狼术 → 4H MACD 0轴过滤"），不重新定义概念

## 一、基本情况

- **交易品种**：GBP/USD
- **交易类型**：日内交易，每天最多一笔
- **分析框架**：4H 定方向 → 1H 定结构 → 15M 定入场
- **入场条件**：三级共振（4H 方向明确 + 1H 价格在中枢边界 + 15M 出现分型 + 背驰）
- **账户**：Dukascopy Bank（每次交易前用户会提供最新净值）
- **交易时段**：伦敦开盘到纽约收盘（北京时间 15:00–01:00）
- **日内强制清仓**：北京时间 01:00 前必须平掉所有仓位，没有例外
- **缠论术语标准**：以 `tradingview/chan-definitions.md` 为准，追溯原典时以缠中说禅原文为准

## 二、缠论指标使用规范

### 图表标注工具（Chan Theory 指标）

**可作为候选参考（需一致性核查，不得无条件采纳）：**
- ⚠️ 笔（顶底分型连线）——指标实现口径可能与缠论原典不同，需目测验证
- ⚠️ 线段（笔的组合）——同上
- ⚠️ 中枢（粉色框）及其 ZD/ZG 价位——作为锚点参考，若明显不合理需指出

**不依赖指标，由我们自己判断：**
- ❌ 买卖点标签（2买、3卖、1买等）——**AI 分析时绝不引用指标的买卖点标签作为判断依据**
- ❌ 背驰判断——MACD 力度对比必须由 AI 手动判断；仅有截图时只能做视觉对比
- ❌ 区间套定位——需要逐级嵌套分析，指标不支持

### 指标设置
- 严格包含-STRICT + 严格延续-STRICT + 宽笔-LOOSE

### 发图规范（用户）
发图时请提供：
1. 三张截图（4H / 1H / 15M），带有指标的笔、线段、中枢标注
2. 各级别中枢的 **ZD 和 ZG 精确价位**（从指标粉色框读取）
3. 若需要精确背驰判断，请额外提供 MACD 数值（DIF/DEA/柱子值）

## 三、风控规则

### 3.1 单笔风险
- 账户净值 < $100,000：单笔最大亏损 = 当前净值 × 4%
- 账户净值 ≥ $100,000：单笔最大亏损 = 当前净值 × 2%

### 3.2 仓位计算
- 公式：手数 = 最大亏损金额 ÷（止损点数 × $10/点）
- 每笔交易前按用户提供的最新账户净值动态计算
- Dukascopy 最小精度 0.01 手
- ⚠️ **入场价敏感性**：实际入场价偏差 ≥ 5 点，必须重新计算手数并提示

### 3.3 日/周纪律
- 每天最多交易 1 笔
- 连续亏损 2 天 → 第 3 天强制观望，不交易；观望一天后计数器归零，第 4 天恢复正常交易
- 三级共振条件不满足 → 不交易

### 3.4 刀口舔血检查
> 定义详见 chan-definitions.md §二十二

入场前必须检查：15M 出现的买卖点方向是否与 4H 大方向一致？
- 若 15M 信号方向与 4H 方向**相反**，且 4H **无衰竭迹象**（无背驰、无盘整背驰） → 该操作 = 刀口舔血 → **不做**
- 在分析结论中必须显式回答此项检查

## 四、盈利加仓策略

### 4.1 加仓触发条件
- 触发前提：浮盈 ≥ 首仓止损点数，且首仓止损已移至成本价
- 持仓盈利后，当 15M 级别走出新的一笔并出现新的买卖点时，用户会把图发过来
- AI 分析确认 15M 新买卖点成立后，再给出加仓建议（加仓手数 + 新止损位）

### 4.2 加仓风控
- 每次加仓时，前序仓位止损必须已移至成本价或盈利位（保本）
- 加仓仓位的独立风险 + 前序仓位的剩余风险 ≤ 当前净值 × 4%（或 2%）
- 加仓方向必须与首仓一致

---

## 五、止盈策略（两段式）

### 5.1 第一止盈：结构目标位
- 以 1H 级别的下一个关键阻力/支撑位（中枢 ZG/ZD、前高前低）作为第一止盈目标
- 到达目标后平仓一半，剩余仓位止损移至成本价
- 不设最低盈亏比门槛，只要缠论结构成立即可入场（当前小资金阶段）

### 5.2 第二止盈：跟结构走
- 剩余仓位跟随缠论结构持有
- 当 1H 或 4H 出现反向信号（顶/底分型确认、趋势背驰、中枢破坏）时平仓
- 硬性截止：无论结构如何，01:00 前必须全部清仓

## 七、分析内容要求

### 7.0 分析前必做

1. **强制读取** `tradingview/chan-definitions.md`
2. 若无法读取，必须明确说明"本次未读取参考文档"，不得伪称已读
3. 每个关键判断标注：课程编号 + 原则名称 + 如何落地到当前图形

### 7.1 常规分析（用户发图时）

> 输出排版模板见 TOOLS.md"交易 / 图表分析执行纪律"章节的固定结构模板。以下仅规定每个级别**必须分析的内容要素**。

**【交易过滤器】**
- **防狼术**（→ 详见 chan-definitions.md §十七）：4H MACD DIF 线在 0 轴上方 → 只做多；0 轴下方 → 只做空
- **刀口舔血检查**（→ 详见 chan-definitions.md §二十二）：15M 信号方向是否与 4H 方向一致？不一致且 4H 无衰竭 → 不做
- ⚠️ 若用户未提供 ZD/ZG：必须说明"中枢未确认，以下为估算，误差约±10-20 点"

**4H 定方向：**
- 分型 → 笔 → 线段 → 中枢 → 走势类型：完整推导
- 背驰判断：仅截图时只做视觉对比，标注"视觉判断，待数值确认"
- MACD 0 轴位置 → 方向结论：做多 / 做空 / 观望
- **中阴识别**（→ 详见 chan-definitions.md §十四）：若 4H 大级别刚完成背驰/盘整背驰，当前可能处于中阴阶段 → 标注"4H 处于中阴，方向待确认"，并判断中阴是否健康

**1H 定结构：**
- 分型 → 笔 → 线段 → 中枢：完整推导，中枢写出 ZD/ZG 计算过程
- 当前价格 vs 中枢位置（上/中/下）
- 是否接近三买/三卖区域
- **奔走型反弹风险**（→ 详见 chan-definitions.md §三十三）：若中枢构造为上下上结构，需评估后上是否仅触及前上低点，提示获利空间可能极小

**15M 定入场：**
- 分型 → 笔 → 线段 → 中枢：完整标注
- 买卖点类型：明确是第几类买卖点，依据是什么
- 背驰确认：哪两段比较？围绕哪个中枢？视觉判断还是数值确认？
- 区间套：用了几重？
- 共振结论：三级共振是否全部满足
- ⚠️ 截图精度无法确认的步骤，明确标注"未确认"
- ⚠️ 多义性情况必须全部列出，不得只给一个答案（→ 详见 chan-definitions.md §十三）

**交易计划（共振满足时）：**
- 入场价 / 止损价 / 第一止盈价（标注预估盈亏比）
- 结构止盈触发条件
- 首仓手数：最大亏损 ÷ 止损点数 ÷ $10
- ⚠️ 入场价敏感性：实际入场价偏差 ≥ 5 点，**必须**重新计算手数并提示
- 📌 数据风控操作
- 📌 OpenClaw 监控价位（加仓关注价 / 止盈关注价）

或：**今天不做** — 原因：具体说明哪个级别、哪个条件不满足

**分析结尾固定输出：**
> 💡 如需审计本次分析，请说：**"审计"**

收到"审计"后：read `tradingview/audit-checklist.md`，逐条对照本次分析输出，标注每条是否满足，不满足的说明原因。

### 7.2 加仓分析
- 15M 新买卖点是否成立 / 建议加仓手数 / 加仓止损位
- 前序仓位止损上移至 / 总仓位风险检查
- 📌 更新关注价位

### 7.3 止盈检查
- 当前结构是否出现反向信号
- 建议：继续持有 / 平仓 / 部分平仓 + 理由
- 📌 如继续持有，更新止盈关注价

### 7.4 结构检查
- 当前结构是否被破坏 / 持仓方向是否仍安全
- 建议：继续持有 / 减仓 / 平仓 + 理由
- 📌 更新关注价位

>
> 【英镑侧】

>

> （填入当天美元侧关键事件）
>
